# Social Publisher Payload Fix Summary

## Issue Description
The social_publisher was receiving the wrong request payload format, causing it to use the instruction text ("Schedule and publish the caption on Instagram.") instead of the actual content generated by content_writer.

### Root Cause
The issue was occurring at multiple levels:
1. **Frontend**: The context object containing content_writer_output was being JSON.stringify'd before sending to the API
2. **Backend**: The agent_routes was only passing the task string to social_publisher, not the full request body
3. **Social Publisher**: Was not properly extracting caption/content from the request structure

## Fixes Applied

### 1. Frontend Fix (api.js)
**File**: `/startup-os/frontend/src/lib/api.js`

**Change**: Modified the payload construction to NOT stringify context for social_publisher:
```javascript
// Enhanced payload with context and session info
// CRITICAL FIX: Don't stringify context for social_publisher - pass it as an object
const enhancedPayload = {
  ...payload,
  // For social_publisher, pass context directly to preserve structure
  context: (agentId === "social_publisher" || agentId === "social-media-publisher") && context 
    ? context 
    : (context ? JSON.stringify(context) : payload.context),
  metadata: {
    ...payload.metadata,
    session_id: currentSession,
  },
};

// For social_publisher, also add caption directly to payload
if ((agentId === "social_publisher" || agentId === "social-media-publisher") && context) {
  if (context.caption) {
    enhancedPayload.caption = context.caption;
  }
  if (context.content_writer_output) {
    enhancedPayload.content = context.content_writer_output;
  }
}
```

### 2. Backend Fix (agent_routes.py)
**File**: `/startup-os/backend/agent_routes.py`

**Change**: Modified to pass the full request body to social_publisher:
```python
# For social_publisher, pass the entire request body to handle caption and context
if agent_id in ["social_publisher", "social-media-publisher", "social_media_publisher"]:
    # Pass the full request body which includes caption, content, and context
    output = await execute_agent_real(agent_id, req.dict(), request_id)
else:
    output = await execute_agent_real(agent_id, task, request_id)
```

### 3. Social Publisher Enhancement
**File**: `/startup-os/backend/social_media_publisher/main_agent.py`

The social_publisher already has logic to extract content from various sources with proper priority:
1. Caption parameter (if provided)
2. Content from state manager (via session_id)
3. Content parameter
4. Task (if it looks like content)

## Expected Behavior After Fix

1. **Frontend sends**:
   - `context` as an object (not stringified) for social_publisher
   - `caption` and `content` fields directly in the payload
   - Proper session_id for state management

2. **Backend receives**:
   - Full request body with all fields
   - Extracts caption/content from the appropriate source
   - Uses the actual generated content, not the instruction

3. **Instagram posts**:
   - The actual caption generated by content_writer
   - With all hashtags and emojis preserved
   - Not the instruction text
